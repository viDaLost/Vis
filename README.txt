Виселица 3D — версия для GitHub Pages с экранной диагностикой ошибок

Структура репозитория должна быть строго такой:

index.html
models/
  Gallows.glb
  peasant.glb
README.txt

Важно:
1. Не загружай папку vis-gh-pages-debug внутрь репозитория как отдельную папку.
2. index.html должен лежать в корне репозитория.
3. Папка models должна лежать рядом с index.html.
4. Имена файлов чувствительны к регистру:
   Gallows.glb — с большой G
   peasant.glb — с маленькой p

Если модели не появились, открой блок «Диагностика» прямо на странице.
Там будет видно:
- загрузился ли Three.js;
- доступен ли WebGL;
- какой HTTP-статус у моделей;
- есть ли 404, ошибка CDN, ошибка декодирования GLB или ошибка WebGL.

После загрузки на GitHub Pages открой сайт в приватной вкладке или очисти кеш Safari.
