Виселица 3D — версия для GitHub Pages

Как правильно загрузить на GitHub Pages:

1. В репозитории должны лежать файлы именно так:

   index.html
   models/Gallows.glb
   models/peasant.glb

2. Не кладите папку Vis-main внутрь другой папки, если GitHub Pages публикует корень репозитория.
   index.html должен быть в корне той папки, которая выбрана в Settings → Pages.

3. Важно сохранить регистр имён файлов:

   Gallows.glb — с большой G
   peasant.glb — маленькими буквами

   GitHub Pages чувствителен к регистру. gallows.glb и Gallows.glb — это разные файлы.

4. После загрузки файлов обновите страницу с очисткой кеша:

   iPhone Safari:
   - открыть страницу
   - нажать обновить
   - если не помогло: Настройки → Safari → Дополнения → Данные сайтов → удалить данные для vidalost.github.io

5. Эта версия использует прямые CDN-ссылки Three.js без importmap:

   https://cdn.jsdelivr.net/npm/three@0.165.0/...

   Так стабильнее на GitHub Pages и мобильных браузерах.

6. Если Three.js или GLB временно не загрузились, игра больше не зависает на экране загрузки.
   Через несколько секунд запускается аварийный DOM-режим, чтобы интерфейс не оставался пустым.

7. Если хотите проверить модели напрямую, откройте в браузере:

   https://ВАШ_ДОМЕН/models/Gallows.glb
   https://ВАШ_ДОМЕН/models/peasant.glb

   Если вместо скачивания/открытия будет 404 — файлы лежат не там или имя написано с другим регистром.
